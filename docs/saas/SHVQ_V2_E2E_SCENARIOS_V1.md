- 작성일: 2026-04-12
- 입력 근거: Claude E2E 시나리오 추출 초안
- 목적: 시스템별 핵심 사용자 흐름을 이관 전/후 동일 품질로 검증

---

## 1) 수량 기준 정리

초안 수량과 운영 관리 수량을 분리해 관리한다.

1. 초안 분류 합계(Raw): `120개` (중복 포함)
2. 운영 KPI(Unique): `95개` (중복 제거 후 관리 기준)
3. 중복 제거 원칙
  - 공통(Cross-cutting)과 시스템 시나리오가 겹치는 항목은 공통으로 1회만 집계
  - 마이그레이션 전용 시나리오는 기능 시나리오와 중복 집계하지 않음

| 마이그레이션 전용 | 4 | Members/Estimate/Activity 이관 + 진도추적 |

## 3) 운영 집계 기준 (Unique 95)

1. 기능 릴리즈 게이트는 `Unique 95`를 기준으로 한다.
2. 시스템 최소 커버리지
  - ERP Core(FMS/PMS/BMS/MAT): 모듈별 최소 10개 이상
  - 나머지 시스템: 영역별 핵심 플로우 전부 포함
3. 고위험 우선순위
  - PJT/견적/MAT, 로그인/권한, Shadow Write 대상 API 연계 플로우

---

## 4) 실행 정책

1. Phase 0
  - 시나리오 ID/사전조건/데이터셋 표준 템플릿 확정
2. Phase 1~2
  - Wave 0~2 기능 중심 회귀(E2E + API 통합)
3. Phase 3~4
  - 병행운영/컷오버 전환 시나리오 집중 검증
4. 배포 게이트
  - P0/P1 실패 0건
  - Critical 경로 성공률 100%

---

## 5) 산출물 형식

1. 기본 컬럼
  - `scenario_id`, `system`, `module`, `precondition`, `steps`, `expected`, `priority`, `owner`
2. 권장 메타
  - `wave`, `related_api`, `related_table`, `is_migration`, `is_shadow_write`
3. 판정
  - `PASS/FAIL/BLOCKED` + 장애 티켓 링크

---

## 6) 오픈 항목

1. Raw 120에서 Unique 95로 축약한 최종 ID 목록 확정
2. 자동화 대상(스모크/야간 회귀) 시나리오 지정
3. 시스템별 담당자(owner) 배정 및 일정 캘린더 반영
